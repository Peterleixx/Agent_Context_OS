# Compile Match Response

## 何时使用

- 收到一个 `任务请求.md`
- 需要基于已有 `PersonaVault` 生成一个 `匹配结果.md`
- 用户想做 `岗位匹配 / 任务匹配 / 角色适配` 的单文件请求返回
- 需要一个保守、带证据、带披露边界的结果，而不是自由聊天

## 不适合使用

- 用户还没有 `PersonaVault`
- 用户想先整理底层资料，而不是生成匹配结果
- 任务需要多轮面试式深聊，而不是一次 Markdown RPC

## 核心目标

把一次请求压缩成稳定流程：

1. 读请求
2. 找卡片
3. 选证据
4. 应用规则
5. 写结果

输出必须可审计、可阅读、可发送。

## 输入契约

确认以下输入：

- `request_md_path`
- `persona_vault_path`
- `response_md_path`
- 场景默认：`role_match`

如果 `request_md_path` 或 `persona_vault_path` 缺失，直接输出结构化失败版 `匹配结果.md`。

## 必须读取的知识库文件

优先读取：

- `01 - Capabilities/能力地图.md`
- 相关能力卡片
- 相关项目卡片
- 相关证据卡片
- `04 - Policies/可说边界.md`
- `04 - Policies/不可说清单.md`
- `04 - Policies/脱敏规则.md`
- `04 - Policies/场景规则-岗位匹配.md`

## 标准流程

1. 读取请求 Markdown
2. 解析请求结构
   - `目标`
   - `必要条件`
   - `加分项`
   - `不希望出现的情况`
   - `披露偏好`
3. 生成匹配维度清单
4. 检索最相关的能力卡片、项目卡片、证据卡片
5. 生成第一版匹配判断
6. 应用披露和脱敏规则
7. 写出固定结构的 `匹配结果.md`
8. 如路径存在，同时写入 `06 - Responses/`
9. 在 `08 - Audit/处理日志.md` 追加一行摘要

## 匹配判断规则

每个维度只能使用以下结论：

- `匹配`
- `部分匹配`
- `暂未证明`
- `不匹配`

### 使用规则

- `匹配`
  - 有明确能力卡片，且至少有 `1` 条直接证据
- `部分匹配`
  - 有能力线索，但证据较弱、较旧，或只覆盖部分要求
- `暂未证明`
  - 没找到足够证据
- `不匹配`
  - 已知信息与要求明显冲突，或出现明确排斥条件

## 证据规则

- 每个 `匹配 / 部分匹配` 条目都必须带证据
- 没有证据时只能写：
  - `暂未证明`
  - `证据不足`
- 不允许把“推测”写成“结论”
- 不允许为了让结果更好看而补写不存在的经历

## 披露规则

始终先读规则文件，再组织输出。

### 优先级

1. `不可说清单.md`
2. `场景规则-岗位匹配.md`
3. `脱敏规则.md`
4. `可说边界.md`

### 处理方式

- 能摘要就摘要
- 需要脱敏就脱敏
- 无法安全输出时，明确写：
  - `不便披露`
  - `无法在当前披露边界下确认`

## 结果文件固定结构

frontmatter 必须包含：

- `protocol_version`
- `request_id`
- `response_type`
- `result`
- `confidence`

正文必须包含：

- `结论`
- `人物画像摘要`
- `匹配拆解`
- `关键证据`
- `已应用的披露限制`
- `待确认问题`

模板见 [templates/匹配结果模板.md](templates/匹配结果模板.md)。

## `result` 与 `confidence` 建议

### `result`

- `match`
  - 大部分必要条件都有证据覆盖
- `partial_match`
  - 有明显匹配点，但也有关键缺口
- `insufficient_data`
  - 资料太薄，无法安全判断
- `no_match`
  - 存在明确不符合项

### `confidence`

- `0.80 - 0.95`
  - 必要条件基本都有证据
- `0.55 - 0.79`
  - 证据部分充分，结论偏保守
- `0.25 - 0.54`
  - 资料不足，只能做弱判断
- `0.00 - 0.24`
  - 明确不匹配或信息极度不足

不要为了显得确定而给高置信度。

## 失败处理

- 请求缺关键字段：
  - 输出结构化失败版 `匹配结果.md`
- `PersonaVault` 缺核心能力卡片：
  - 输出“资料不足，无法判断”
- 证据都被披露规则拦截：
  - 输出“无法在当前披露边界下完成判断”

## 输出风格

- 偏保守
- 偏结构化
- 偏证据导向
- 不做销售式吹捧
- 不写成长篇动机信

## 完成检查

完成前确认：

- 正文 6 个 section 全部存在
- 每个“匹配 / 部分匹配”条目都能追到证据
- 已应用的披露限制明确写出来
- 如果资料不足，已经明确降级为保守结论
- 输出可以直接作为单文件回调发送
